Help & Setup Guide

Everything you need to get TrackCast working with Spotify and OBS.

Quick Start

  1. Create a free Spotify Developer app and copy its Client ID and Secret.
  2. Turn on OBS WebSocket (Tools menu inside OBS Studio).
  3. Run this app and follow the wizard. Done.

Step 1 · Create a Spotify Developer App

  1. Go to developer.spotify.com/dashboard and sign in with your Spotify account.
  2. Click Create app. Name and description can be anything.
  3. In Redirect URIs add exactly: http://localhost:8888/callback
  4. Save. Open the app, then click Settings to copy your Client ID and Client Secret.

Step 2 · Enable OBS WebSocket

  1. Open OBS Studio.
  2. Top menu: Tools → WebSocket Server Settings.
  3. Check Enable WebSocket server.
  4. Set a server password and note it down. Default port is 4455.
  5. Click OK.

Step 3 · Add the text source

In the OBS step of the wizard, after the connection test passes, click "Check if it exists". If the source does not exist yet, the app can create it for you in the current scene. You can then drag, resize, and style it from inside OBS.

Customizing the overlay text

The format field accepts two placeholders:

  • {trackName} — current track title
  • {artistName} — artist or comma-separated artists

Example: ♫ {trackName} by {artistName}

Troubleshooting

OBS source does not update
Confirm the source name in OBS matches the one configured in the app exactly, including capitalization.
OBS connection fails
Check that OBS WebSocket is enabled, the password matches, and Windows Firewall is not blocking port 4455.
Spotify authorization fails
Verify the redirect URI in your Spotify Developer dashboard is exactly http://localhost:8888/callback, no trailing slash.
"Token expired" keeps appearing
The app auto-refreshes tokens 5 minutes before expiry. If errors persist, go to Settings → Spotify and re-authorize.
App icon stays grey in tray
Polling is paused, no Spotify track is playing, or the Spotify connection dropped. Resume polling from the sidebar footer.

Logs & reporting bugs

Detailed logs are stored at %APPDATA%\TrackCast\logs\main.log. Include the last 100 lines when reporting an issue on GitHub.
